# RFC 10117: AI Tool Affordances

## Summary

Define affordances and constraints for AI-driven tools so behavior is predictable and safe.

## Motivation

- Clear boundaries reduce surprises.
- Structured outputs enable better integrations.

## Proposal

- Tool surface conventions (names, args, outputs).
- Steering patterns and “next action” guidance.
- Safety constraints for file mutation.

## Open questions

- What should be stable vs versioned/proposed?
- How do we test the tool surface contract?
