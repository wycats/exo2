---
title: Product patterns
description: Mockup-derived product UI patterns for documentation examples.
---

Product examples should feel like cockpit fragments inside the documentation:
dark panels, visible workflow state, compact rails, and explicit labels. These
patterns mirror the product sheet in the mockups rather than the earlier light
card examples.
