# Boundaries and Scope of This Exosuit

This file captures what *does not* belong in this repository, and how we keep the setup portable without turning this into a dumping ground for every project.

It is a first-pass sketch of boundaries to be revisited as the workspace grows.

## 1. Scope of This Repo (Outline)

- Canonical personal workspace for me plus AI
- What kinds of artifacts are expected to live here

## 2. What Stays Outside (Outline)

- Code projects that should live in their own repositories
- Data or information that belongs in other systems (e.g. company tools, private notes)

## 3. Portability of Tooling (Outline)

- Using MCP servers, VS Code configuration, or other mechanisms to share patterns without sharing this repo
- How we think about reusing values/patterns across other environments

## 4. Re-evaluating Boundaries (Outline)

- Signs the repo is accreting too much
- How we decide to spin something out into its own place
